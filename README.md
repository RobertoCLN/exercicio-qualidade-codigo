## 📌 Sobre o Projeto

Este repositório demonstra comparações práticas entre código de má qualidade e código de boa qualidade, abordando critérios fundamentais de engenharia de software.

Cada pasta contém:
- Um exemplo de código ruim
- Um exemplo de código melhorado
- Um README explicando as melhorias aplicadas

## 🎯 Objetivo

Demonstrar na prática conceitos como:

- Legibilidade
- Documentação
- Padronização
- Reutilização
- Manutenibilidade
- Robustez
- Testabilidade
- Eficiência
- Escalabilidade
- Segurança
