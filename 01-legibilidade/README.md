# 01 - Legibilidade

## ❌ Problema no código ruim

- Nomes de variáveis não são descritivos.
- Nome da função não indica o que ela faz.
- Não há comentários explicando o objetivo.
- O código exige análise detalhada para entender sua intenção.

## ✅ Por que o código bom é melhor?

- A função possui nome descritivo.
- As variáveis são claras e autoexplicativas.
- Existe documentação explicando propósito e parâmetros.
- O código pode ser entendido rapidamente sem esforço excessivo.

## 🛠 O que foi aplicado para melhorar?

- Uso de nomes significativos.
- Comentários em formato JSDoc.
- Estrutura mais organizada.
- Melhor legibilidade geral.
