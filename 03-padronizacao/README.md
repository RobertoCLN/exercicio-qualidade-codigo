# 03 - Padronização

## ❌ Problema no código ruim

- Uso de convenções inconsistentes (caixa alta desnecessária).
- Mistura de estilos de nomenclatura.
- Uso de "var" em vez de "const" ou "let".
- Falta de tipagem explícita no retorno.
- Formatação e indentação inadequadas.

## ✅ Por que o código bom é melhor?

- Segue padrão camelCase do TypeScript.
- Usa "const" corretamente.
- Define tipo de retorno.
- Aplica boas práticas de organização e clareza.
- Mantém consistência visual e estrutural.

## 🛠 O que foi aplicado para melhorar?

- Aplicação de convenções da linguagem.
- Uso correto de escopo (const).
- Melhor formatação e organização.
- Padronização de nomenclatura.
