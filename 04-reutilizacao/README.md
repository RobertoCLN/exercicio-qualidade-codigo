# 04 - Reutilização

## ❌ Problema no código ruim

- A mesma lógica foi duplicada em múltiplas funções.
- Viola o princípio DRY (Don't Repeat Yourself).
- A manutenção se torna difícil caso a regra mude.

## ✅ Por que o código bom é melhor?

- A lógica foi centralizada em uma única função.
- O código pode ser reutilizado em diferentes partes do sistema.
- Facilita manutenção e evolução do sistema.

## 🛠 O que foi aplicado para melhorar?

- Aplicação do princípio DRY.
- Modularização da lógica.
- Redução de duplicação de código.
